# Capa — Behavioral Capability Intelligence Engine

**Capa** is a deterministic source-code intelligence engine that infers what a software engineer can actually _do_ — not what they claim — by analyzing their real open-source behavior.

Instead of resumes, questionnaires, or self-reported skills, Capa extracts verifiable engineering capabilities directly from GitHub repositories and converts them into a multi-dimensional **Capability Vector**. A Generative-AI reasoning layer then interprets these vectors to provide hiring insights, team composition, career guidance, and technical interview generation.

Capa is designed around a strict principle:

> The AI never decides the truth.
> The deterministic engine computes the truth.
> The AI only explains it.

This separation makes the system explainable, auditable, and resistant to hallucinations.

---

## Why Capa Exists

Hiring engineers today relies on:

- resumes
- keyword matching
- coding tests
- self-reported experience

All of these are unreliable signals.

Capa instead analyzes:

- dependency graphs
- project structure
- architecture patterns
- behavioral commit activity
- capability correlations across repositories

The result is a **behavioral technical profile** grounded in real work rather than claims.

## System Architecture

Capa is a dual-layer system:

### 1. Deterministic Intelligence Engine (Rust)

Responsible for factual analysis and capability extraction.

### 2. Generative Reasoning Layer (Python + Gemini)

Responsible for explanation and decision interpretation.

The AI layer cannot fabricate skills because it receives only structured evidence generated by the Rust engine.

## How It Works

### Step 1 — Repository Analysis

The Rust engine ingests a GitHub user and extracts signals from:

- dependency manifests (Cargo.toml, package.json, requirements.txt, etc.)
- file naming patterns
- folder structure
- language composition
- commit behavior
- architecture indicators

### Step 2 — Capability Inference

Signals are combined into weighted channels:

| Signal               | Reliability |
| -------------------- | ----------- |
| Dependencies         | Very High   |
| File structure       | High        |
| Project layout       | High        |
| Keywords             | Medium      |
| Activity patterns    | Low         |
| Language composition | Weak        |

The engine computes calibrated scores and produces a stable capability vector.

### Step 3 — Similarity & Indexing

Developers are indexed and compared using cosine similarity in capability space.

### Step 4 — AI Interpretation

The Python pipeline sends structured JSON evidence to Gemini, which:

- explains strengths
- evaluates job fit
- builds teams
- suggests career growth
- generates interview questions

## Rust Deterministic Core

All commands below run without AI.

### Data Ingestion

Fetch repositories and analyze signals:

```
cargo run -- --verify
TARGET_USERNAME=darkraider01 cargo run
```

### Calibration & Analysis

```
cargo run -- --recalibrate
cargo run -- --analyze-distribution
cargo run -- --evaluate-archetypes
cargo run -- --similarity-matrix
```

## AI Generative Pipeline

The AI pipeline reads structured JSON output from Rust and performs reasoning.

### 1. Zero-Shot Natural Language Search

Translates non-technical recruiter language directly into structured capability bounds, pulling the required domains directly from the system registry.

```bash
python ai_pipeline.py --nl-search "Looking for a seasoned rust developer with strong systems design background"
```

_(Note: Full Rust `--search` indexing is currently in development; the AI gateway will successfully translate the structural query matrix for preview)._

### Developer Intelligence Report

```
python ai_pipeline.py --profile darkraider01 --explain
```

### Similarity Explanation

```
python ai_pipeline.py --similar darkraider01 --explain
```

### Job-Fit Evaluation

```
python ai_pipeline.py --evaluate-fit darkraider01 "Backend engineer with scalable API and cloud experience"
```

### Career Growth Plan

```
python ai_pipeline.py --growth-plan darkraider01
```

### Team Builder

```
python ai_pipeline.py --build-team "Real-time collaborative coding platform"
```

### AI Technical Interviewer

```
python ai_pipeline.py --generate-interview darkraider01
```

## Environment Configuration

Create `.env`:

```
DATABASE_URL=postgresql://user:password@localhost:5432/capability_search
GITHUB_TOKEN=github_pat_...
TARGET_USERNAME=darkraider01
GEMINI_API_KEY=your_key_here
```

If `GEMINI_API_KEY` is omitted, the system runs in offline deterministic mode.
